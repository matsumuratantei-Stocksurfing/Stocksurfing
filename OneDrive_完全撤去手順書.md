# OneDrive 完全撤去手順書

> Windows 11 から OneDrive を完全に削除し、Windows Update でも復活しないようにする手順

⚠️ **このセッション（Cowork チャット）を終了してから実施してください。**
セッション中はファイルをロックしているため、フォルダのリダイレクト解除が失敗します。

---

## 撤去前の状態確認

現在の松村さん環境（2026/05/08 時点）：
- ✅ OneDrive **アプリ本体は既にアンインストール済み**（スタートメニューに出ない）
- ⚠️ ただし **C:\Users\matsu\OneDrive\ フォルダは残っており**、Windows のドキュメント・ピクチャがそこにリダイレクトされている
- ⚠️ Windows Update で OneDrive アプリが**復活する可能性**がある

つまり、本作業の目的は：
1. ドキュメント・ピクチャのリダイレクトを **C:\Users\matsu\Documents\** などのローカルに戻す
2. OneDrive フォルダの中身を C:\matsumura\ などに退避
3. C:\Users\matsu\OneDrive\ フォルダを削除
4. レジストリで再インストールを禁止

---

## ステップ1：Cowork セッションを終了する

Claude デスクトップアプリを **完全に終了** してください：

1. Claude アプリを閉じる（×ボタン）
2. タスクトレイ（隠れたアイコン）にClaudeが残っていないか確認、あれば「終了」
3. タスクマネージャーで Claude が動いていないことを確認

これで OneDrive\ドキュメント\Claude\... 内のファイルロックが解除されます。

---

## ステップ2：重要ファイルのバックアップ

エクスプローラーで以下のフォルダを **コピー** して C:\matsumura\backup_onedrive_20260508\ に保存：

- C:\Users\matsu\OneDrive\ドキュメント\Claude\
- C:\Users\matsu\OneDrive\ドキュメント\GitHub\（後でクローンを再構築するので無くても可）
- C:\Users\matsu\OneDrive\ドキュメント\ にある他の重要ファイル
- C:\Users\matsu\OneDrive\ピクチャ\ にある写真
- C:\Users\matsu\OneDrive\ にある PDF やJSONファイル

**手順：**
1. エクスプローラーで C:\Users\matsu\OneDrive\ を開く
2. 中身全て選択（Ctrl+A）
3. コピー（Ctrl+C）
4. C:\matsumura\backup_onedrive_20260508\ を新規作成
5. 貼り付け（Ctrl+V）
6. コピー完了まで待つ

---

## ステップ3：ドキュメントのリダイレクトを解除

1. エクスプローラーの左サイドバー「ドキュメント」を **右クリック**
2. **プロパティ**
3. **「場所」タブ**
4. 現在のパス（C:\Users\matsu\OneDrive\ドキュメント）を確認
5. **「標準に戻す(R)」** ボタンをクリック
6. 「元の場所へリダイレクトしますか？」→ **「はい」**
7. 「フォルダー内の既存のすべてのファイルを新しい場所に移動しますか？」→ **「はい」**（ステップ2でバックアップ済みなので安全）
8. ファイル移動完了まで待つ
9. **「OK」** で閉じる

エラー「同じ場所にリダイレクトできないフォルダーがあります」が出る場合：
- これは「ピクチャ」も同じ場所にリダイレクトされている可能性
- ピクチャも同じ手順で「標準に戻す」
- それぞれ独立した場所（Documents / Pictures）に戻る

---

## ステップ4：ピクチャのリダイレクトを解除

ステップ3と同じ手順を「ピクチャ」フォルダで実施。

---

## ステップ5：デスクトップのリダイレクトを確認

「デスクトップ」フォルダも OneDrive 配下になっていないか確認：

1. デスクトップ（左サイドバー）を右クリック → プロパティ → 場所
2. パスが `C:\Users\matsu\Desktop` ならOK
3. `C:\Users\matsu\OneDrive\デスクトップ` ならステップ3と同じ手順で標準に戻す

---

## ステップ6：OneDrive フォルダを削除

リダイレクト解除完了後：

1. エクスプローラーで C:\Users\matsu\OneDrive\ を開く
2. 中身が空（または不要なファイルのみ）なことを確認
3. C:\Users\matsu\OneDrive\ フォルダ自体を **右クリック → 削除**
4. ゴミ箱からも削除（Shift+Delete で完全削除も可）

---

## ステップ7：レジストリで OneDrive 再インストールを禁止

⚠️ レジストリ編集は慎重に。スクショ通りの手順を厳密に。

**7-1. レジストリエディタを起動**

1. Win + R → `regedit` と入力 → Enter
2. 「このアプリがデバイスに変更を加えることを許可しますか？」→ **「はい」**

**7-2. 該当キーに移動**

左のツリーで以下を順に展開：

```
HKEY_LOCAL_MACHINE
  └ SOFTWARE
    └ Policies
      └ Microsoft
        └ Windows
```

**7-3. OneDrive キーを作成（無ければ）**

`Windows` を **右クリック → 新規 → キー** → 名前を **「OneDrive」** にする

**7-4. 値を作成**

`OneDrive` キーを選択した状態で、右ペインで **右クリック → 新規 → DWORD（32ビット）値**

- 名前：**`DisableFileSyncNGSC`**
- 値のデータ：**`1`**（ダブルクリックして編集）

これで OneDrive の自動同期が完全に禁止されます。

**7-5. レジストリエディタを閉じる**

---

## ステップ8：PCを再起動

設定を反映させるため、Windows を再起動。

再起動後：
- ✅ スタートメニューで「OneDrive」検索 → 出てこない
- ✅ ドキュメントフォルダの実体が `C:\Users\matsu\Documents\`
- ✅ ピクチャフォルダの実体が `C:\Users\matsu\Pictures\`
- ✅ Windows Update してもOneDrive は復活しない

## トラブル時

### Q. ステップ3で「アクセスが拒否されました」と出る
→ どこかのアプリ（Claude、Edge、エクスプローラー等）がそのフォルダを使用中。すべてのアプリを閉じて再試行。

### Q. リダイレクト解除後も C:\Users\matsu\OneDrive\ が消えない
→ 中身を空にしてから削除。フォルダ自体は残っていても害はないが、気持ち悪ければ手動削除。

### Q. レジストリ編集が怖い
→ ステップ7はスキップしてもOK。ただし、その場合 Windows Update 後に OneDrive が復活する可能性あり。復活したら再度アンインストール。

### Q. ステップ6で「OneDrive アプリ本体」が逆に復活してしまった
→ 設定 → アプリ → インストール済みアプリ → OneDrive → アンインストール
